## Manual

- Review role behavior, rendered templates, shell scripts, and operational workflow changes manually or through role validation tasks until Molecule coverage is introduced.
- Review project-agnostic wording manually so shared roles, docs, tests, examples, and comments do not mention consuming projects.
- Toolbox Bash scripts must source `/opt/toolbox/lib/helpers.sh` through `../lib/helpers.sh` and use underscore-prefixed helper functions such as `_info`, `_warn`, `_error`, and `_cmd` for operator-facing logging and command execution. Use `_warn` like `_info` for attention-worthy non-fatal messages. Keep command selection separate from operation helpers: prefer `_cmd _docker_postgres psql` over combined helper names such as `_pg_psql_cmd`. Use `_cmd` for commands whose output is not captured and that are not used as conditions; do not use `_cmd` for interactive commands, command substitutions, condition checks, or report payload commands where prefixed output would change the script contract.
